Per

A minimal cubical CCHM implementation of MLTT-80, Martin-Löf Intuitionistic Type Theory. Per is an internalized model capable of checking theorems about itself, providing a foundational core for higher-order dependently typed systems.

Abstract

Per is a theorem prover that constitutes a MLTT-80 core for a dependently-typed lambda calculus. It is specifically constrained to exclude pattern matching, let-bindings, and implicit arguments to focus on the pure mathematical properties of the core. It encompasses two types of universes as in HTS, dependent products Pi, dependent pairs Sigma, identity types Id, and 0, 1, 2, W types for well-founded definitions.

Unlike full CCHM cubical HIT-enriched systems with complex composition structures, Per remains a minimal cubical system. It focuses on correctness, soundness, totality, and canonicity — the essential attributes for formal mathematics and self-internalized type theory.

Canonicity of Natural Numbers

Comparison of constructive natural numbers: structural vs. homotopical representations.

Strong Canonicity

canonical.per — Natural numbers as a direct W-type. Every closed term of type Nat normalizes to a constructor form, perfectly satisfying the canonicity property.

def A : U := 𝟐
def B (b : 𝟐) : U₀ := ind₂ (λ (_ : 𝟐), U₀) 𝟎 𝟏 b
def Nat : U₀ := W (b : A), B b

def zero : Nat := sup A B 0₂ (λ (e : 𝟎), ind₀ Nat e)
def succ (n : Nat) : Nat := sup A B 1₂ (λ (u : 𝟏), n)

Standard constructive semantics with strict reduction paths.

Homotopical Canonicity

homotopical.per — Homotopy-inductive natural numbers. Lacks the strong canonicity of structural types as they are defined via transp and PathP.

def ℕ := W (x : 𝟐), ℕ-ctor x
def zero : ℕ := sup 𝟐 ℕ-ctor 0₂ (ind₀ ℕ)

def 𝟎⟶ℕ (C : ℕ → U) (f : 𝟎 → ℕ) : C zero → C (sup 𝟐 ℕ-ctor 0₂ f)
 := transp (<i> C (sup 𝟐 ℕ-ctor 0₂ (λ (x : 𝟎), ind₀ (PathP (<_> ℕ) 
    (ind₀ ℕ x) (f x)) x @ i))) 0

Demonstrates the fluidity of semi-cubical foundations without strict structural constraints.

Self-Checking Core

Internalized Model

Per is designed to be self-hosting in its logic. The core constructs are represented within the language itself, allowing for proofs about the typechecker's own behaviors and properties.

def internalizing : MLTT-73
 := ( Pi ★, Π-lambda ★, Π-apply ★, Π-β ★, Π-η ★,
      Sigma ★, pair ★, pr₁ ★, pr₂ ★, Σ-β₁ ★, Σ-β₂ ★, Σ-η ★,
      Path-1 ★, idp-1 ★, J-1 ★, J-β ★,
      ★
    )

Foundational Focus

Per excludes high-level sugar to maintain a lean, provable core. It leverages Judgmental Equality and Strong Normalization, ensuring that every well-typed term reduces to a unique normal form, even in the presence of universes and well-founded types.

MLTT and CCHM Foundations

Fibrational System

Leveraging PathP for deep-layered identity proofs. Per's foundations enable the definition of contractible types, propositions, and sets, forming a basis for univalent formalization.

def Pi (O : 𝟏) (A : U) (B : A → U) : U := Π (x : A), B x
def Π-lambda (O : 𝟏) (A : U) (B : A → U) (b : Pi ★ A B) : Pi ★ A B := λ (x : A), b x
def Π-apply (O : 𝟏) (A: U) (B: A → U) (f: Pi ★ A B) (a: A) : B a := f a
def Π-β (O : 𝟏) (A : U) (B : A → U) (a : A) (f : Pi ★ A B) : Path (B a) (Π-apply ★ A B (Π-lambda ★ A B f) a) (f a) := idp (B a) (f a)
def Π-η (O : 𝟏) (A : U) (B : A → U) (a : A) (f : Pi ★ A B) : Path (Pi ★ A B) f (λ (x : A), f x) := idp (Pi ★ A B) f
def Sigma (O : 𝟏) (A : U) (B : A → U) : U := summa (x: A), B x
def pair (O : 𝟏) (A: U) (B: A → U) (a: A) (b: B a) : Sigma ★ A B := (a, b)
def pr₁ (O : 𝟏) (A: U) (B: A → U) (x: Sigma ★ A B) : A := x.1
def pr₂ (O : 𝟏) (A: U) (B: A → U) (x: Sigma ★ A B) : B (pr₁ ★ A B x) := x.2
def Σ-β₁ (O : 𝟏) (A : U) (B : A → U) (a : A) (b : B a) : Path A a (pr₁ ★ A B (a ,b)) := idp A a
def Σ-β₂ (O : 𝟏) (A : U) (B : A → U) (a : A) (b : B a) : Path (B a) b (pr₂ ★ A B (a, b)) := idp (B a) b
def Σ-η (O : 𝟏) (A : U) (B : A → U) (p : Sigma ★ A B) : Path (Sigma ★ A B) p (pr₁ ★ A B p, pr₂ ★ A B p) := idp (Sigma ★ A B) p

Identity System

Leveraging PathP for deep-layered identity proofs. Per's foundations enable the definition of contractible types, propositions, and sets, forming a basis for univalent formalization.

def transport (A B: U) (p : PathP (<_> U) A B) (a: A): B := transp p 0 a
def singl (A: U) (a: A): U := Σ (x: A), Path A a x
def eta (A: U) (a: A): singl A a := (a, idp A a)
def contr (A : U) (a b : A) (p : Path A a b) : Path (singl A a) (eta A a) (b, p) := <i> (p @ i, <j> p @ i /\ j)
def trans_comp (A : U) (a : A) : Path A a (transport A A (<i> A) a) := <j> transp (<_> A) -j a
def subst (A : U) (P : A -> U) (a b : A) (p : Path A a b) (e : P a) : P b := transp (<i> P (p @ i)) 0 e
def subst-comp (A: U) (P: A → U) (a: A) (e: P a) : Path (P a) e (subst A P a a (idp A a) e) := trans_comp (P a) e
def D (A : U) : U₁ := Π (x y : A), Path A x y → U
def J (A: U) (x: A) (C: D A) (d: C x x (idp A x)) (y: A) (p: Path A x y)
  : C x y p := subst (singl A x) (\ (z: singl A x), C x (z.1) (z.2)) (eta A x) (y, p) (contr A x y p) d
def J-1 (O : 𝟏) (A : U) (x : A) (C: D A) (d: C x x (idp A x)) (y: A) (p: Path A x y)
  : C x y p := subst (singl A x) (\ (z: singl A x), C x (z.1) (z.2)) (eta A x) (y, p) (contr A x y p) d

Homotopy Type Theory

univalent.per — Implementation of the core of Homotopy Type Theory, providing path spaces, identity systems, and the J-induction principle within the semi-cubical framework.

def Path (A : U) (x y : A) : U := PathP (<_> A) x y
def idp (A : U) (x : A) : Path A x x := <_> x
def singl (A: U) (a: A): U := Σ (x: A), Path A a x
def eta (A: U) (a: A): singl A a := (a, idp A a)
def sym (A: U) (a b : A) (p : Path A a b) : Path A b a := <i> p @ -i
def contr (A : U) (a b : A) (p : Path A a b) : Path (singl A a) (eta A a) (b, p) := <i> (p @ i, <j> p @ i /\ j)
def isContr (A: U) : U := Σ (x: A), Π (y: A), Path A x y
def isContrSingl (A : U) (a : A) : isContr (singl A a) := ((a,idp A a),(\ (z:singl A a), contr A a z.1 z.2))
def cong (A B : U) (f : A → B) (a b : A) (p : Path A a b) : Path B (f a) (f b) := <i> f (p @ i)
def ap (A: U) (a x: A) (B: A → U) (f: A → B a) (b: B a) (p: Path A a x) : Path (B a) (f a) (f x) := <i> f (p @ i)

DNF Solver for Homotopy Cubes

The CCHM (Cubical Computational Higher Type Theory) model requires a robust solver for face lattice operations. Per implements a simple DNF (Disjunctive Normal Form) solver to handle boundary conditions in hcomp and transp operations. This solver is essential for ensuring that composition across different faces remains consistent and that judgmental equality between complex face expressions is decidable.

Accessing Higher Dimensions
